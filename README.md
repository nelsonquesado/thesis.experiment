# A Method for Formulating and Assessing Public Transport Interventions Based on Equity Principles

This repository contains the replication code, transit datasets, and output figures for the doctoral thesis research conducted at the Federal University of Ceará (UFC). 

The project provides a computational framework for the ex-ante assessment of public transport interventions. It evaluates spatial accessibility, network resilience, and vulnerability, specifically contrasting utilitarian optimization approaches with equity-focused strategies (e.g., maximin) for urban mobility planning.

## Repository Structure

The project is structured into data processing, scenario generation, and comparative analysis using R.

### 💻 R Scripts (`/R/`)
The scripts are numbered sequentially to reflect the workflow from data preparation to final evaluation:
* `0. land use scenarios.R`: Prepares socioeconomic data and simulates land-use scenarios (optimist/pessimist) using `aopdata`.
* `1. gtfs reduction.R`: Filters and reduces baseline GTFS networks (bus and metro) for computational efficiency during the routing process.
* `2. baseline scenario.R`: Evaluates the current state of the transport network.
* `3. do nothing scenario.R`: Simulates future outcomes assuming no interventions are made.
* `4. pre optimal gtfs.R`: Prepares the network structures for the optimization algorithms.
* `5. subsidy scenarios.R`: Models the impact of different financial subsidy structures on the transport system.
* `6. fare optimal.R`: Optimizes transit fares for system efficiency.
* `7. timetable optimal.R`: Optimizes transit frequencies and timetables using standard utilitarian objectives.
* `8. timetable optimal - maxmin.R`: Optimizes timetables focusing on equity, maximizing the minimum accessibility for vulnerable groups.
* `9. comparing scenarios v3.R`: Generates summary statistics and aggregated comparisons across all modeled scenarios.
* `10. ex ante.R`: Performs the final ex-ante framework assessment combining accessibility, equity, and resilience metrics.

### 🚌 Transit Data (`/data/gtfs/`)
Contains the General Transit Feed Specification (GTFS) files representing the public transport networks for various scenarios.
* **Baselines**: `1.b bus.baseline.zip`, `3.a metro.baseline.full.zip`, `3.b metro.baseline.zip`

* **Optimized Networks**: `5. bus.pre.optimal.zip`, `6. bus.optimal.zip`, `6. bus.optimal - maximin.zip`

## Prerequisites & Dependencies

The scripts are written in **R**. To reproduce the analysis, you will need to install the following core packages:

* `tidyverse` 
* `sf` 
* `r5r` 
* `GTFSwizard` 
* `aopdata` 
* `data.table` 
* `GA` 

**Note on memory allocation:** The routing algorithms process highly dense travel time matrices using `r5r`. Ensure your R session is configured with sufficient memory for Java before loading the package (e.g., `options(java.parameters = "-Xmx6G")`), as specified in the scripts.

## Usage

1. Clone or download this repository.
2. Ensure your working directory is set to the root of the project.
3. Run the scripts in their numbered order (`0` through `10`). 
4. Outputs will be saved automatically into the `figs/` directory and intermediate data components will be routed appropriately.
